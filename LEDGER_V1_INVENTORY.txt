AUTH FLOW (Firebase Phone Auth)
================================================================
1. User enters phone (+254 XXX XXX XXX)
2. Firebase sends OTP via SMS
3. User enters 6-digit code
4. On verify: checks Firestore for existing user profile
   - Profile exists → go to /home
   - No profile → go to /profile-setup
5. Profile setup writes to Firestore users/{uid}
6. Firestore rules require request.auth != null

Providers:
  authServiceProvider    → AuthService instance
  authStateProvider      → StreamProvider<User?> (Firebase Auth state)
  currentUserProvider    → Provider<User?> (current Firebase User)
  userProfileProvider    → FutureProvider.family<UserModel?, uid>
  phoneAuthProvider     → PhoneAuthHandler (sendOTP, verifyOTP)
